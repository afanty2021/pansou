## 引言
本文档详细阐述了项目中缓存键的生成规则和序列化机制的实现。系统通过精心设计的缓存键生成策略，确保了缓存键的唯一性和可预测性，有效避免了键冲突。同时，采用高效的序列化机制，优化了缓存数据的存储和传输性能。这些设计共同构成了系统高性能缓存体系的核心。

## 缓存键生成规则

### 缓存键生成策略
系统采用多参数组合的缓存键生成策略，通过`GenerateCacheKey`函数将搜索关键词、频道列表、源类型和插件列表等参数组合生成唯一的缓存键。该策略确保了不同搜索条件产生不同的缓存键，避免了缓存冲突。

```mermaid
flowchart TD
Start([开始]) --> NormalizeKeyword["标准化关键词<br/>转小写、去空格"]
NormalizeKeyword --> CheckChannels["检查频道列表"]
CheckChannels --> |为空| UseAllChannels["使用预计算的all_channels哈希"]
CheckChannels --> |不为空| SortChannels["排序频道列表"]
SortChannels --> |小于5个| JoinChannels["直接连接字符串"]
SortChannels --> |大于等于5个| HashChannels["计算MD5哈希"]
HashChannels --> CacheChannels["存入channelHashCache"]
JoinChannels --> ChannelsResult["返回频道哈希结果"]
HashChannels --> ChannelsResult
UseAllChannels --> ChannelsResult
ChannelsResult --> CheckSourceType["检查源类型"]
CheckSourceType --> |为tg| SetPluginsHash["插件哈希设为none"]
CheckSourceType --> |不为tg| GetPluginsHash["获取插件列表哈希"]
SetPluginsHash --> BuildKeyString["构建缓存键字符串"]
GetPluginsHash --> BuildKeyString
BuildKeyString --> CalculateMD5["计算MD5哈希"]
CalculateMD5 --> ReturnKey["返回最终缓存键"]
```

### 参数处理机制
系统对各个参数进行标准化处理，确保相同语义的参数生成相同的缓存键。关键词经过小写化和去除首尾空格处理；频道和插件列表经过排序处理，消除参数顺序对缓存键的影响；源类型为空时默认为"all"。

### 哈希优化策略
为了提高性能，系统实现了哈希值缓存机制。对于常用的频道列表和插件列表，系统预先计算其哈希值并缓存，避免重复计算。同时，对于小型列表（少于5个元素），直接使用排序后的字符串连接作为哈希，减少计算开销。

## 序列化机制实现

### 序列化器设计
系统采用接口驱动的序列化器设计，定义了`Serializer`接口，支持多种序列化实现。当前主要使用`GobSerializer`，同时也保留了`JSONSerializer`以保持向后兼容性。


### Gob序列化实现
`GobSerializer`使用Go语言内置的gob包进行序列化，具有较高的性能和紧凑的二进制格式。通过`sync.Pool`实现缓冲区对象池，减少了内存分配开销，提高了序列化性能。

```mermaid
sequenceDiagram
participant Client as "客户端"
participant Serializer as "GobSerializer"
participant Buffer as "缓冲区池"
Client->>Serializer : Serialize(v)
Serializer->>Buffer : Get() 获取缓冲区
Serializer->>Serializer : 重置缓冲区
Serializer->>Serializer : 创建gob编码器
Serializer->>Serializer : 执行编码
Serializer->>Serializer : 复制结果数据
Serializer->>Buffer : Put() 归还缓冲区
Serializer-->>Client : 返回序列化数据
```

## 性能影响分析

### 缓存体积优化
Gob序列化格式相比JSON等文本格式更加紧凑，显著减少了缓存数据的存储空间。对于包含大量搜索结果的响应，这种体积优化效果尤为明显，有助于提高缓存命中率和降低存储成本。

```mermaid

A[原始数据] --> B[Gob序列化]
A --> C[JSON序列化]
B --> D["体积: 较小<br/>效率: 较高"]
C --> E["体积: 较大<br/>效率: 较低"]
```

### 读写性能评估
序列化和反序列化操作对缓存的读写性能有直接影响。`GobSerializer`通过缓冲区池减少了内存分配，提高了处理速度。在高并发场景下，这种优化能够显著降低GC压力，保持系统稳定。

### 缓存键生成开销
缓存键生成涉及字符串处理、排序和哈希计算等操作。系统通过预计算常用列表的哈希值、缓存已计算的哈希结果以及对小型列表采用轻量级处理策略，有效控制了键生成的开销。

## 结论

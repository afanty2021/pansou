## 简介

PanSou 是一个高性能的网盘资源搜索API服务，支持通过Telegram频道和自定义插件进行资源搜索。本指南旨在帮助新手用户快速在本地或容器环境中部署并运行 PanSou 服务，从克隆仓库到成功发起第一次搜索的完整流程。

## 环境准备

在部署 PanSou 服务之前，需要确保系统环境满足基本要求。根据选择的部署方式（Docker 或源码编译），准备相应的环境。

### Docker 部署环境

- 安装 Docker 引擎（版本 19.03 或更高）
- 安装 Docker Compose（推荐使用 v2.x 版本）
- 确保系统有足够的磁盘空间用于缓存（建议至少 1GB）

### 源码编译部署环境

- Go 语言环境（版本 1.18 或更高）
- Git 工具（用于克隆仓库）
- 可选：Node.js 环境（仅当需要使用 MCP 服务时）

## 部署方式

PanSou 提供了两种主要的部署方式：使用 Docker 快速部署和从源码编译部署。用户可以根据自身需求和环境选择合适的方式。

### Docker 部署

Docker 部署是最简单快捷的方式，适合大多数用户。

#### 前后端集成版

使用以下命令一键启动服务：

```bash
docker run -d --name pansou -p 80:80 -e ENABLED_PLUGINS="labi,zhizhen,shandian,duoduo,muou,wanou" ghcr.io/fish2018/pansou-web
```

#### 纯后端API版

使用 Docker Compose 部署（推荐）：

```bash
# 下载配置文件
curl -o docker-compose.yml https://raw.githubusercontent.com/fish2018/pansou/refs/heads/main/docker-compose.yml

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f
```

### 源码编译部署

对于希望自定义配置或进行二次开发的用户，可以选择从源码编译部署。

1. 克隆仓库：

```bash
git clone https://github.com/fish2018/pansou.git
cd pansou
```

2. 构建项目：

```bash
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -ldflags="-s -w -extldflags '-static'" -o pansou .
```

3. 运行服务：

```bash
./pansou
```

## 服务启动

无论选择哪种部署方式，启动服务后都可以通过配置的端口访问 PanSou 服务。

### 环境变量配置

PanSou 支持通过环境变量进行配置，常用的环境变量包括：

| 环境变量 | 描述 | 默认值 |
|----------|------|--------|
| **PORT** | 服务端口 | `8888` |
| **PROXY** | SOCKS5代理 | 无 |
| **CHANNELS** | 默认搜索的TG频道 | `tgsearchers3` |
| **ENABLED_PLUGINS** | 指定启用插件 | 无 |

## API使用

PanSou 提供了简洁的 API 接口，支持 POST 和 GET 两种请求方式。

### 搜索API

**接口地址**：`/api/search`  
**请求方法**：`POST` 或 `GET`

#### POST请求示例

```json
{
  "kw": "速度与激情",
  "channels": ["tgsearchers3", "xxx"],
  "conc": 2,
  "refresh": true,
  "res": "merge",
  "src": "all",
  "plugins": ["jikepan"],
  "cloud_types": ["baidu", "quark"],
  "ext": {
    "title_en": "Fast and Furious",
    "is_all": true
  }
}
```

#### GET请求示例

```bash
curl "http://localhost:8888/api/search?kw=速度与激情&channels=tgsearchers3,xxx&conc=2&refresh=true&res=merge&src=tg&cloud_types=baidu,quark&ext={\"title_en\":\"Fast and Furious\",\"is_all\":true}"
```

## 常见问题排查

在部署和使用过程中可能会遇到一些常见问题，以下是排查建议。

### 端口冲突

如果启动服务时提示端口已被占用，可以修改 `PORT` 环境变量或停止占用端口的进程。

### 依赖缺失

- Docker 部署：确保 Docker 和 Docker Compose 已正确安装
- 源码编译：确保 Go 环境已正确配置，且 `go mod download` 能够正常执行

### 插件未启用

确保在环境变量中正确设置了 `ENABLED_PLUGINS`，多个插件用逗号分隔。

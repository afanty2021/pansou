## SearchRequest 搜索请求

`SearchRequest` 结构体定义了客户端发起搜索请求时所需的所有参数。该结构体通过 `json` 标签定义了其在JSON序列化和反序列化时的字段名称，确保了API接口的稳定性和兼容性。

| 字段名 | 数据类型 | JSON标签 | 业务约束 | 作用 |
| :--- | :--- | :--- | :--- | :--- |
| `Keyword` | string | `kw` | `binding:"required"` | 搜索的关键词，是必填项，用于指定搜索内容。 |
| `Channels` | []string | `channels` | 可选 | 指定要搜索的Telegram频道列表，为空时使用配置中的默认频道。 |
| `Concurrency` | int | `conc` | 可选 | 指定并发搜索的协程数量，用于控制搜索速度和资源消耗。 |
| `ForceRefresh` | bool | `refresh` | 可选 | 强制刷新标志，为 `true` 时将忽略缓存，直接从源获取最新数据。 |
| `ResultType` | string | `res` | 可选 | 指定返回结果的类型，可选值包括 `all` (返回所有)、`results` (仅返回results)、`merge` (仅返回merged_by_type)。 |
| `SourceType` | string | `src` | 可选 | 指定数据来源类型，可选值包括 `all` (全部来源)、`tg` (仅Telegram)、`plugin` (仅插件)。当 `src=tg` 时会忽略 `plugins` 参数。 |
| `Plugins` | []string | `plugins` | 可选 | 指定要搜索的插件列表，为空时搜索所有启用的插件。当 `src=plugin` 时会忽略 `channels` 参数。 |
| `Ext` | map[string]interface{} | `ext` | 可选 | 扩展参数，一个键值对映射，用于向特定插件传递自定义参数，如 `{"title_en": "The Matrix"}`。 |
| `CloudTypes` | []string | `cloud_types` | 可选 | 指定要返回的网盘类型列表，如 `["baidu", "tianyi"]`，为空时返回所有类型的链接。 |

## SearchResponse 搜索响应

`SearchResponse` 结构体定义了搜索服务返回给客户端的完整响应数据。它是一个容器，包含了搜索的总数量、结果列表以及按网盘类型合并的链接。

| 字段名 | 数据类型 | JSON标签 | 业务约束 | 作用 |
| :--- | :--- | :--- | :--- | :--- |
| `Total` | int | `total` | 必填 | 搜索结果的总数。当 `ResultType` 为 `merged_by_type` 时，此值为所有合并链接的总数；否则为 `Results` 列表的长度。 |
| `Results` | []SearchResult | `results,omitempty` | 可选 | 一个 `SearchResult` 对象的切片，包含详细的搜索结果。当 `ResultType` 为 `results` 或 `all` 时包含数据
